# 缓存策略文档

## 缓存层级

### L1: 应用内存缓存
- 配置数据
- 字典数据

### L2: Redis 缓存
- 用户会话
- 热点数据
- API 响应

## 缓存策略

### 用户会话
- Key: `session:{user_id}`
- TTL: 24 小时
- 数据: 用户基本信息

### 热点帖子
- Key: `post:{id}`
- TTL: 1 小时
- 数据: 帖子详情

### 帖子列表
- Key: `posts:list:{page}:{sort}`
- TTL: 5 分钟
- 数据: 帖子列表

### 排行榜
- Key: `rank:posts:hot`
- TTL: 10 分钟
- 数据: 热门帖子 ID 列表

## 缓存更新策略

### Cache Aside
1. 查询先查缓存
2. 缓存未命中查数据库
3. 写入缓存
4. 更新时删除缓存

### 缓存预热
- 启动时预热热门数据
- 定时任务刷新排行榜

## 缓存穿透防护
- 空值缓存（TTL 5 分钟）
- 布隆过滤器

## 缓存雪崩防护
- 随机 TTL
- 热点数据永不过期
