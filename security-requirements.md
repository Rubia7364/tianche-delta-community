# 安全需求文档

## 用户认证

### 认证方式
- **JWT 认证**
  - Token 有效期：24 小时
  - Refresh Token 有效期：7 天
  - Token 存储：localStorage（前端）+ Redis（后端）

### 密码安全
- 使用 bcrypt 加密
- Salt rounds：10
- 密码强度要求：至少 8 位，包含字母和数字

## 权限控制

### RBAC 模型

**角色定义：**
- `USER`：普通用户
  - 发帖、评论、组队
  - 查看个人数据
  
- `MODERATOR`：版主
  - USER 的所有权限
  - 删除违规帖子
  - 封禁用户（临时）
  
- `ADMIN`：管理员
  - MODERATOR 的所有权限
  - 用户管理
  - 打手审核
  - 系统配置

## 敏感数据加密

- **密码**：bcrypt 加密，不可逆
- **支付信息**：AES-256 加密
- **个人隐私信息**：脱敏展示（手机号、邮箱）

## API 安全

### API 限流策略

- **全局限流**：1000 req/s
- **单 IP 限流**：100 req/s
- **登录接口**：5 次/分钟
- **注册接口**：3 次/分钟
- **发帖接口**：10 次/小时

### SQL 注入防护

- 使用 Prisma ORM
- 参数化查询
- 禁止拼接 SQL

### XSS/CSRF 防护

- 输入验证（class-validator）
- 输出转义（React 自动转义）
- CSRF Token（POST/PUT/DELETE 请求）

## HTTPS 要求

- 强制 HTTPS
- TLS 1.2+
- 自动重定向 HTTP → HTTPS

## 数据备份

- 数据库每日备份
- 备份保留 30 天
- 异地备份

## 审计日志

记录以下操作：
- 用户登录/登出
- 权限变更
- 敏感数据访问
- 订单交易
- 用户封禁/解封
